# 功能：浏览器扩展（Chrome Extension）

> 状态：规划完成 / 待开发
> 创建日期：2026-06-05
> 关联功能：[[AI自动分类]] [[浏览器插件-书签管理]]

---

## 一、概述

为 MarkBox 开发 Chrome 浏览器扩展（Manifest V3），让用户在浏览网页时可以一键收藏书签，无需离开当前页面。支持快速收藏 Popup + 侧边栏管理 Side Panel 的混合交互模式。

---

## 二、技术选型

| 决策点 | 选择 | 理由 |
|---|---|---|
| **开发框架** | WXT (`wxt.dev`) | 支持 React + TypeScript，热重载，Manifest V3 原生支持，开发体验接近 Next.js。可直接复用项目现有的 shadcn/ui 组件体系 |
| **目标浏览器** | Chromium 系（Chrome / Edge） | 覆盖绝大多数中文用户，Manifest V3 标准 |
| **跨浏览器** | 暂不考虑 Firefox / Safari | 先打磨一个平台的体验，后续可用 WXT 的跨平台能力低成本扩展 |
| **认证方式** | A：共享 Session Cookie（默认） + B：API Key（fallback） | 用户在 MarkBox 主站已登录后，插件通过同域 Cookie 自动获得认证，零配置。设置页生成 API Key 作为备用方案，方便高级用户和自部署场景 |
| **交互模式** | Popup（快速收藏）+ Side Panel（智能管理）混合 | Popup 用于"看一眼、一键收藏"；Side Panel 用于不离开页面浏览和管理书签库 |
| **UI 体系** | 复用主站 shadcn/ui + Tailwind CSS + motion 动画 | 保持品牌一致性，减少重复开发 |

## 三、交互设计

### 3.1 Popup — 快速收藏

用户点击工具栏图标 → 弹出 400×560px 的 Popup 窗口。

**流程：**

```
点击图标
  → 自动检测当前页面 URL
  → 显示加载骨架屏（~0.5s）
  → 元数据提取（标题、封面、描述、站点名、内容类型）
  → AI 分析完成（标签 + 分类 + 摘要）
  → 展示预览卡片（信息完整填充）
  → 用户一键点击「确认收藏」
  → 成功动画 + Toast 提示
  → Popup 自动关闭
```

**预览卡片展示内容：**
- 封面图（顶部横幅）
- 标题（可点击编辑）
- 站点名 + Favicon
- AI 标签（彩色标签，可点击删除/添加）
- AI 分类路径（可点击修改）
- AI 摘要（2-3 句，折叠显示）
- 状态选择（默认"待读"）

**编辑能力：**
- 标题：点击进入编辑模式
- 标签：点击 × 删除，点击 + 添加新标签
- 分类：点击切换分类选择器
- 摘要：点击可修改

**特殊场景：**

1. **已收藏提示：** 如果当前页面 URL 已存在于书签库中，Popup 顶部醒目提示「已收藏 ✅」+ 显示收藏时间 + 当前状态，下方提供「更新书签」和「打开详情」按钮

2. **批量收藏建议：** 当检测到页面内存在明显的内容列表（B 站合集、博客系列目录等），在 Popup 底部展示「此页面还有 3 篇相关内容，是否批量收藏？」提示条。点击后展开批量选择界面

3. **加载失败降级：** 元数据提取或 AI 分析失败时，展示降级 UI —— 仅保留 URL 输入框 + 标题手动输入 + 「仅保存链接」按钮

---

### 3.2 Side Panel — 智能管理

用户右键页面 → 「在 MarkBox 中打开」或通过快捷键 → 浏览器右侧滑出 400px 宽的侧边栏。

**基础功能（A 层 — 微型控制台）：**

- **最近收藏列表**：按时间倒序展示最近 20 条书签
- **搜索框**：输入关键词即时搜索全局书签
- **当前页面状态指示**：显示当前标签页是否已收藏，收藏时间和状态
- **快速切换状态**：每条书签支持一键切换（待读→在读→已读→归档）
- **「打开主站」链接**：跳转到 MarkBox 完整页面

**智能上下文（C 层 — 场景洞察）：**

当浏览某个网页时，Side Panel 自动分析：

1. **关联书签推荐：** 「你收藏过 3 篇相似内容」——基于标签和内容相似度，展示关联书签卡片
2. **知识缺口提示：** 「这篇内容和你收藏的《XXX》观点相反」——当检测到主题相同但来自不同立场的文章
3. **阅读路径引导：** 「AI 建议按此顺序阅读：A → B → C」——将相关内容串联成学习路径
4. **重复内容检测：** 「你已收藏过类似内容」——避免重复收藏同一主题

**状态呈现：**
- 首次加载：显示「最近收藏」列表
- 检测到关联内容：在列表顶部插入「关联推荐」卡片组
- 无关联内容：仅显示搜索 + 最近列表

### 4.2 认证方案

**方案 A：共享 Session Cookie（默认，零配置）**

```
用户在主站 (markbox.app) 已登录
  → Chrome 插件可以访问同域 Cookie
  → API 请求自动携带 Session Token
  → 后端识别用户身份
  → 插件无需额外登录步骤
```

实现细节：
- 插件 manifest 声明 `host_permissions: ["https://markbox.app/*"]`
- API 请求使用 `credentials: "include"`
- 后端 CORS 配置允许插件来源

**方案 B：API Key（备用）**

```
用户在设置页面生成 API Key
  → 在插件中手动填入（或扫码同步）
  → 插件存储 API Key 到 chrome.storage.local
  → 每次请求在 Authorization header 中携带
  → 适合自部署场景或跨域通信
```

两种方案自动检测：
1. 插件启动时先尝试方案 A（检查 cookie 有效性）
2. 方案 A 失败 → 提示用户使用方案 B
3. 方案 B 的 API Key 存储在加密的 chrome.storage 中

### 4.3 消息通信

```
Popup ←→ Background Service Worker
  - Popup 通过 runtime.sendMessage 请求收藏操作
  - Background 处理后端通信和存储

Side Panel ←→ Background Service Worker ←→ Content Script
  - Side Panel 通过 Background 获取当前页面 URL
  - Content Script 可选注入以获取页面内容列表（批量收藏）

Content Script → Background Service Worker
  - 检测页面内链接列表（hFeed / article list）
  - 回传批量候选 URL 给 Popup
```

### 4.4 API 通信

插件复用现有 MarkBox 后端 API：

| 操作 | API 端点 | 方法 |
|---|---|---|
| 收藏书签 | `/api/bookmarks` | POST |
| 搜索书签 | `/api/search?q=` | GET |
| 获取最近列表 | `/api/bookmarks?limit=20` | GET |
| 检查 URL 是否已收藏 | `/api/bookmarks?q=<url>` | GET |
| 更新书签状态 | `/api/bookmarks/[id]` | PATCH |
| 获取标签列表 | `/api/tags` | GET |
| 获取分类树 | `/api/categories` | GET |
| AI 分类 | `/api/ai/categorize` | POST |
| 元数据提取 | `/api/metadata` | POST |

### 4.5 元数据提取优化

对于当前浏览页面，**优先直接从 DOM 提取**（无需经过后端 HTTP 抓取）：

```
Chrome 插件可直接访问当前页面 DOM
  → 读取 OG 标签、JSON-LD、<title>
  → 读取 Open Graph 图片
  → 获取页面 Favicon
  → 构建元数据对象
  → 仅在失败时 fallback 到后端 API
```

优势：速度更快（0 延迟，无需网络请求），不依赖后端抓取。

## 五、UI/UX 设计要点

### 5.1 视觉一致性

- 复用主站 Warm Editorial 设计语言 + ESG 调色板
- 浅色/深色主题跟随 `prefers-color-scheme`，与浏览器主题同步
- 卡片样式、标签配色、动画曲线全部复用
- 磨砂玻璃效果（`backdrop-blur`）和 SVG 噪声纹理保持一致

### 5.2 动画

- **收藏成功**：卡片缩略动画飞入 Popup 底部 → 绿色勾 + 脉冲效果
- **加载态**：骨架屏微光扫过动画（shimmer）
- **已收藏提示**：黄色星标旋转弹入
- **标签编辑**：标签弹性缩放进入/退出
- **侧栏展开**：内容从左滑入，带轻微的 spring 弹性

### 5.3 快捷键

| 快捷键 | 操作 |
|---|---|
| `Ctrl+Shift+D` | 一键收藏当前页面（不弹出 Popup，静默收藏） |
| `Ctrl+Shift+S` | 打开 Side Panel |
| `Ctrl+Shift+F` | 聚焦搜索框 |

### 5.4 离线与错误处理

- **无网络**：缓存最近收藏列表，允许本地浏览
- **后端不可用**：显示离线标识，操作进入队列，待恢复后同步
- **AI 超时**：降级到仅保存 URL + 标题，标签留空待后续补充
- **认证过期**：引导用户重新登录主站或配置 API Key
