# 功能：知识图谱视图

> 状态：规划完成 / 待开发
> 创建日期：2026-06-05
> 关联功能：[[AI自动分类]] [[图表可视化]] [[时间线视图]]

---

## 一、概述

为 MarkBox 新增「知识图谱」视图，将用户的整个书签库可视化为可交互的节点网络图。用户可以直观看到自己的知识结构、兴趣版图以及书签之间的隐藏关联。支持从宏观知识集群深入到单条书签的三层渐进探索，并叠加时间维度查看知识图谱的演变。

---

## 二、技术选型

| 决策点 | 选择 | 理由 |
|---|---|---|
| **图节点模型** | 分层混合：内层标签/分类（骨架）+ 外层书签（叶子） | 既有宏观知识结构，又能深入细节。初始展示干净不杂乱，展开后信息丰富 |
| **图布局算法** | 力导向图（Force-Directed Graph） | 视觉冲击力强，交互自然，相关节点自动聚拢，一眼看到兴趣版图。答辩演示效果好 |
| **渲染库** | `react-force-graph-2d` | 基于 D3 forceSimulation 的 React 封装，Canvas 渲染性能好（数百节点不卡），支持节点拖拽/缩放/自定义渲染。未来可升级 3D 版本做答辩炫酷演示 |
| **交互模式** | 三层渐进探索 + 时间轴滑块 | 第一层知识集群 → 第二层书签围绕展开 → 第三层详情抽屉。底部时间轴查看知识演变 |
| **数据来源** | 本地计算 + 现有 API | 标签关联度、分类层级数据均已存在，无需新 API。书签列表通过 `/api/bookmarks` 获取 |
| **动画系统** | Canvas 力导向动画 + React 详情面板 motion 动画 | 力导向图用 Canvas 原生动画保证性能；详情面板用 motion/react 保持与其他视图一致的动画语言 |

## 三、交互设计

### 3.1 第一层：知识星云（初始视图）

**展示内容：**
- 分类节点（大圆，带名称标签，颜色区分）
- 标签节点（中圆，围绕在关联分类周围）
- 节点大小 = 该书签数量
- 连线粗细 = 关联强度（共享书签数量）
- 同名标签和分类之间自然聚合，形成 5-10 个"知识集群"

**用户可操作：**
- **拖拽节点**：拨动节点，力导向图自动重新平衡
- **缩放**：滚轮缩放，看到更多细节或全局概览
- **悬停节点**：tooltip 显示「标签名 + 书签数 + 关联标签数」
- **点击节点**：进入第二层
- **双击空白处**：重置视图（回到第一层默认状态）
- **框选**：按住 Shift 拖动，可框选多个节点并高亮对应书签

**视觉设计：**
- 深色背景下节点用柔和发光效果（类似星系星云质感）
- 不同分类用不同的柔和色调（ESG 调色板延展）
- 连线用半透明细线，避免遮挡节点
- 集群自动居中，初始视图保证所有节点可见

**顶部工具栏：**
- 搜索框：输入标签名 / 分类名 / 书签标题 → 高亮对应节点，闪烁动画
- 筛选器：按时间范围、标签、分类筛选可见节点
- 「重置视图」按钮
- 「截图导出」按钮 → 导出当前图谱为 PNG（答辩分享用）

### 3.2 第二层：书签展开

点击第一层的标签/分类节点后：

**动画效果：**
- 被点击节点放大并移至视图中心
- 周围弹出行星级轨道，相关书签节点沿轨道出现
- 书签节点 = 小型缩略卡片（favicon + 标题前 8 字截断）
- 节点颜色与所属内容类型对应（视频=红、文章=蓝、仓库=绿）
- 同一标签的书签自动聚拢在标签节点周围

**书签节点操作：**
- **悬停**：tooltip 显示完整标题 + 标签 + 收藏时间
- **点击**：进入第三层（详情抽屉）
- **拖拽**：可单独拖拽书签节点，不影响整体布局
- **多人关联书签**：如果同一书签同时属于多个标签，它会位于两个标签节点之间，同时连接两边（用虚线表示跨标签关联）

**返回操作：**
- 点击空白处或「← 返回星云」按钮回到第一层

### 3.3 第三层：详情抽屉

点击第二层的书签节点后：

从右侧滑出一个详情抽屉（复用主站 `BookmarkDetailSheet` 组件的样式和动画）：

- 封面图（或 favicon 兜底）
- 标题 + 原始链接
- AI 摘要
- 标签列表（可点击跳转到该标签的第一层视图）
- 收藏时间 + 阅读状态
- 「打开原网页」按钮
- 「编辑书签」按钮

关闭抽屉 → 回到第二层视图。

### 3.4 时间轴维度（C 层）

图谱底部有一个水平时间轴滑块：

- **范围：** 从最早收藏时间到最新收藏时间
- **滑块：** 可拖动选择时间窗口
- **播放按钮：** 自动播放知识图谱随时间演变的过程
  - 节点逐步出现（按收藏时间顺序）
  - 新标签节点从小变大
  - 连线逐步建立
  - 知识集群从无到有地"生长"
- **播放速度：** 1× / 2× / 5×
- **当前时间标签：** 显示正在查看的时间范围（如「2025年 3月 - 6月」）

**答辩演示脚本：**
> "这是我从 2025 年 3 月开始的知识图谱。点击播放，可以看到我的知识版图如何从最初的「前端入门」逐步扩展到「React 生态」「后端」「AI」——这记录了我的学习路径变迁。"

## 四、技术实现

### 4.1 数据结构

**图谱数据格式（前端计算生成）：**

```typescript
interface GraphData {
  nodes: GraphNode[];
  links: GraphLink[];
}

interface GraphNode {
  id: string;                    // 唯一标识
  type: 'category' | 'tag' | 'bookmark';
  label: string;                 // 显示名称
  size: number;                  // 节点半径 = 书签数量 * 系数
  color: string;                 // 节点颜色
  bookmarkCount?: number;        // 关联书签数（category/tag 节点）
  bookmarkId?: string;           // 书签 ID（bookmark 节点）
  parentIds?: string[];          // 所属标签/分类 ID 列表
  contentType?: string;          // 内容类型（bookmark 节点）
  favicon?: string;              // favicon URL（bookmark 节点）
  coverImage?: string;           // 封面图 URL
  createdAt?: string;            // 收藏时间（时间轴用）
  // 力导向图状态（由 react-force-graph 管理）
  x?: number;
  y?: number;
  vx?: number;
  vy?: number;
}

interface GraphLink {
  source: string;               // 源节点 ID
  target: string;               // 目标节点 ID
  strength: number;             // 关联强度（共享书签数量，影响连线粗细）
  type: 'tag-tag' | 'tag-category' | 'bookmark-tag';
}
```

**数据生成流程：**

```
获取所有书签（/api/bookmarks?limit=1000）
  → 提取每个书签的标签列表和分类
  → 构建标签节点的共现矩阵
    - 两个标签出现在同一书签中 → 建立连线
    - 连线 strength = 共同书签数量
  → 构建分类-标签父子关系
  → 分类/标签节点 size = 关联书签总数
  → 书签节点（第二层按需加载）
```

### 4.2 组件树

```
KnowledgeGraphView (主视图容器)
├── GraphToolbar（顶部工具栏）
│   ├── SearchInput（搜索框 + 高亮跳转）
│   ├── FilterDropdown（时间/标签/分类筛选）
│   ├── ResetButton（重置视图）
│   └── ScreenshotButton（截图导出）
├── ForceGraph（react-force-graph-2d 主画布）
│   └── CustomNodeCanvas（自定义 Canvas 节点渲染）
│       ├── CategoryNode（分类大圆 + 标签 + 书签计数）
│       ├── TagNode（标签中圆 + 标签）
│       └── BookmarkNode（书签小卡片 + favicon + 标题截断）
├── NodeTooltip（悬停 tooltip，在 Canvas 上层用 HTML overlay）
├── BookmarkDetailSheet（第三层详情抽屉，复用现有组件）
└── TimelineSlider（底部时间轴）
    ├── PlayPauseButton（播放/暂停）
    ├── SpeedSelector（1x/2x/5x）
    └── TimeRangeLabel（当前时间范围指示）
```

### 4.3 核心交互状态

```
appState = {
  viewLayer: 1 | 2 | 3,        // 当前视图层级
  selectedNode: string | null,  // 当前选中节点 ID
  expandedNode: string | null,  // 当前展开的标签/分类节点（第二层）
  timeRange: [Date, Date],      // 时间轴范围
  isPlaying: boolean,           // 时间轴播放状态
  playbackSpeed: 1 | 2 | 5,    // 播放速度
  searchQuery: string,          // 搜索关键词
  highlightedNodeIds: string[], // 高亮节点列表
}
```

### 4.4 性能考虑

| 场景 | 策略 |
|---|---|
| **节点数 < 200** | 正常运行，60fps，全量渲染 |
| **节点数 200-500** | 降低力导向模拟精度，使用 Canvas 实例回收 |
| **节点数 > 500** | 默认显示仅分类/标签节点（第一层），书签节点按需展开。超过一定距离的连线自动隐藏 |
| **时间轴播放** | 预计算每帧的节点状态，用 RAF 驱动，避免每帧重新计算力导向 |

### 4.5 API 需求

知识图谱主要使用现有 API，无需新增后端端点：

| 用途 | API | 说明 |
|---|---|---|
| 获取全量书签 | `GET /api/bookmarks?limit=1000` | 添加 `includeTags=true` 参数获取标签数据 |
| 获取标签列表 | `GET /api/tags` | 含计数，用于构建标签节点 |
| 获取分类树 | `GET /api/categories` | 含计数和层级，用于构建分类节点 |

**可选新增 API：**

| 端点 | 说明 |
|---|---|
| `GET /api/graph/connections?bookmarkId=xxx` | 获取与指定书签关联的其他书签（基于标签重叠 + 语义相似度） |

## 五、UI/UX 设计要点

### 5.1 视觉风格

- **星云主题：** 节点在深色画布上呈现柔和发光效果，类似星系星云
- **调色板：** 在 ESG 调色板基础上扩展，为不同知识领域分配柔和区分色
  - 技术/编程：蓝紫色系
  - 设计/创意：暖橙色系
  - 学术/理论：绿色系
  - 工具/效率：灰色系
- **连线：** 半透明细线（`rgba(255,255,255,0.15)` 或对应主题色），不抢夺节点视觉权重
- **节点光晕：** 使用 Canvas 径向渐变模拟光晕效果

### 5.2 动画

- **初始加载：** 节点从中心散开，力导向模拟逐步稳定（~3s 达到平衡态）
- **节点展开（1→2 层）：** 中心节点光晕扩大，书签节点从中心弹出并绕行
- **详情抽屉（2→3 层）：** 与主站一致的右侧滑入动画
- **时间轴播放：** 节点按时间顺序出现，带有"弹入"的 spring 效果
- **搜索高亮：** 目标节点闪烁脉冲动画（3 次），吸引注意
- **节点拖拽：** 手指/鼠标松开后节点带有惯性漂移，逐渐减速

### 5.3 响应式

- Canvas 自动适应容器大小
- 移动端/触屏：支持双指缩放和单指拖拽
- 小屏设备（< 768px）：默认进入简化模式（仅显示第一层）

## 六、开发阶段

### 阶段 1：基础图渲染
- 数据计算层（标签共现矩阵 + 节点大小计算）
- react-force-graph-2d 集成
- 第一层节点渲染（分类 + 标签）
- 连线渲染
- 基础交互（拖拽、缩放、悬停 tooltip）

### 阶段 2：三层交互
- 点击展开第二层（书签节点弹出）
- 第三层详情抽屉
- 搜索高亮 + 节点定位
- 返回和重置操作

### 阶段 3：时间轴
- 时间轴滑块 + 数据过滤
- 播放动画（节点按时间出现/消失）
- 速度控制

### 阶段 4：打磨
- Canvas 自定义节点渲染（光晕、书签卡片样式）
- 动画曲线优化
- 性能优化（大规模节点）
- 截图导出
- 移动端适配

## 七、扩展可能

- **3D 知识图谱**：用 `react-force-graph-3d` 升级为 Three.js 3D 星云，答辩神级演示
- **对比图谱**：和其他用户的知识图谱对比（如果后续有社交功能）
- **自动发现主题**：AI 自动检测图谱中的"知识孤岛"和"隐性关联"
- **学习路径可视化**：在图谱上标注推荐阅读顺序
