> 创建日期：2026-06-05
> 关联功能：[[AI自动分类]] [[知识图谱视图]] [[浏览器扩展]] [[仪表盘]] [[书签详情]]

---

## 一、概述

为 MarkBox 新增两大协同功能：「关联推荐」和「学习路径生成」。关联推荐基于标签共现和语义向量双重匹配，在书签详情、Dashboard 和独立发现页面三个层次呈现相关内容；学习路径在此基础上，用 AI 将推荐结果编排成有序的阅读路线，支持手动调整、进度追踪和微型笔记，最终可导出为 Markdown 学习总结。

两个功能形成完整闭环：推荐回答"还有什么值得看"，学习路径回答"按什么顺序看"，进度追踪回答"学到了什么"。

---

## 二、技术选型

| 决策点 | 选择 | 理由 |
|---|---|---|
| **推荐算法** | C：标签共现协同过滤（召回） + AI 语义向量匹配（精排）混合 | 标签共现瞬时响应、零 API 调用，适合快速召回候选集；语义向量发现表面标签不同但内容相关的内容。冷启动靠标签，数据积累后语义优势递增 |
| **向量存储** | 书签表新增 `embedding` JSON 文本字段 | 项目使用 libsql（SQLite），不支持 pgvector。个人书签管理器规模（<10k 条），应用层计算余弦相似度即可，无需专用向量数据库 |
| **向量生成模型** | DeepSeek V4 / text-embedding 兼容接口 | 复用现有 `deepseek.ts` 客户端（openai SDK），无需引入新依赖。嵌入维度 1024-1536，JSON 序列化后约 4-8KB/条 |
| **推荐呈现层次** | 三层渐进：详情页 → Dashboard → 独立发现页 | 详情页推荐开发量最小、触发最自然；Dashboard 推荐增强首页体验；发现页作为答辩亮点展示算法深度 |
| **学习路径生成** | C：AI 自动骨架 + 手动拖拽 + 知识缺口检测 | AI 做难度分析、类型标注和初始排序，用户保留完全控制权。知识缺口检测是答辩差异化亮点——"不只是整理，更是学习规划" |
| **路径可视化** | D：路线图主视图 + 知识图谱叠加切换 | 路线图清晰展示顺序和进度（日常使用）；图谱叠加展示路径在知识版图中的位置（答辩演示）。两视图共享数据，一键切换 |
| **进度追踪** | C：进度条 + 里程碑庆祝 + 微型笔记链 + 导出 | 笔记链让路径不只是"读完就忘"，答辩时可以说"这不是冷冰冰的收藏夹，而是知识内化工具" |
| **行为反馈** | 推荐点击 + 路径完成数据回传 | 用户点击推荐或完成路径节点时记录行为，用于优化推荐排序和难度评估 |

### 模块 A：关联推荐

#### A.1 算法设计

**第一层 — 标签共现召回（本地计算，毫秒级）：**

```
给定书签 B，其标签集合 T = {t1, t2, t3}

步骤 1：找到所有与 T 中任意标签关联的书签 → 候选集 C
步骤 2：计算每篇候选书签的关联分数：
  score = Σ(共享标签数) / (候选书签的标签总数)
步骤 3：排除已收藏的重复 URL
步骤 4：取 Top-20 作为召回候选集

特殊情况：
  - 书签只有 1 个标签 → 放宽匹配，匹配该标签 + 同分类下其他标签
  - 候选集不足 5 个 → 扩展到同分类下热门书签补充
```

**第二层 — 语义向量精排（AI 增强，后端计算）：**

```
步骤 1：对召回候选集（20 条）和当前书签计算 Embedding 向量
步骤 2：余弦相似度排序
步骤 3：过滤相似度 < 0.7 的低相关结果
步骤 4：返回 Top-5 推荐结果

优化策略：
  - Embedding 向量在书签创建/更新时异步生成并缓存，推荐时直接读取
  - 向量生成去重：相同 URL 不重复计算
  - 批量更新：新增书签后批量计算与最近 N 条书签的相似度
```

**第三层 — 行为反馈优化（持续学习）：**

```
记录用户行为：
  - 推荐展示次数
  - 点击次数
  - 从推荐中收藏的次数

反馈到排序：
  - 被多次展示但从未点击的标签组合 → 降权
  - 被点击并收藏的组合 → 加权
  - 跨领域意外发现（serendipity）→ 保留 10% 的随机扰动避免"信息茧房"
```

#### A.2 推荐呈现层次

**层次 1 — 书签详情页推荐（基础层）：**

在 `BookmarkDetailSheet` 底部新增「相关推荐」区块：

- 展示 3-5 篇推荐书签的横向滚动卡片行
- 每张卡片：封面缩略 + 标题 + 匹配标签高亮 + 相似度百分比
- 点击卡片 → 关闭当前详情 → 打开推荐书签的详情（形成"无限探索"体验）
- 在详情抽屉中按 `→` 方向键快速跳转到第一篇推荐

**层次 2 — Dashboard 个性化推荐（增强层）：**

在 `DashboardView` 顶部新增「为你推荐」区块：

- 基于用户整体收藏画像（高频标签、主导分类、最近阅读趋势）
- 展示 3 个维度的推荐：
  - 「继续深入」：基于最近 7 天收藏的主题，推荐深度内容
  - 「拓展边界」：基于核心兴趣的相邻领域（如学 React 的推荐 Vue 对比文章）
  - 「重温旧藏」：推荐收藏超过 30 天但标记为"待读"的高质量书签（基于 AI 评分）
- 用 Tabs 切换三个维度

**层次 3 — 独立「发现」页面（探索层）：**

新增 `DiscoverView` 组件，作为独立视图入口（在侧边栏或顶部导航）：

- **主题漫游模式**：从一个书签出发，沿关联推荐不断跳转，形成"阅读轨迹"。可视化当前浏览路径（面包屑 + 节点图）
- **跨领域意外发现**：AI 推荐与当前兴趣完全不同的高质量内容（"你一直在看前端，试试这篇设计心理学的文章？"）
- **热门趋势**：基于全平台（如果后续有社交功能）或个人的收藏趋势变化
- **每日精选**：AI 从书签库中每日推荐一篇"今日必读"

### 模块 B：学习路径生成

#### B.1 路径生成流程

**步骤 1：用户输入**

用户可以通过以下方式创建学习路径：
- **手动创建**：命名路径 + 选择目标标签/分类（如"系统学习 Rust"）
- **从书签快捷创建**：在某篇书签的详情页点击「加入学习路径」→ 选择已有路径或新建
- **AI 推荐创建**：在 Dashboard 或发现页，AI 检测到用户收藏了多篇同一主题的书签时主动提示——"你已收藏了 8 篇 Rust 相关内容，是否生成学习路径？"

**步骤 2：AI 分析（自动）**

AI 扫描用户书签库中与目标主题相关的所有书签，分析每篇：

```typescript
interface PathItemAnalysis {
  bookmarkId: string;
  difficulty: 'beginner' | 'intermediate' | 'advanced';
  contentType: 'tutorial' | 'documentation' | 'project' | 'article' | 'video' | 'paper';
  estimatedMinutes: number;      // 预估阅读时长
  prerequisites: string[];       // 前置知识点
  keyTakeaways: string;          // AI 生成的核心收获（1 句话）
}
```

**步骤 3：骨架生成（AI 自动）**

AI 根据分析结果生成初始路径骨架：

```
路径：系统学习 Rust
├── Day 1-2：入门基础
│   ├── 《Rust 官方入门指南》[tutorial, beginner, 45min]
│   ├── 《B 站：30 分钟搞懂所有权》[video, beginner, 30min]
│   └── 《Rust by Example》[documentation, beginner, 60min]
├── Day 3-5：核心概念
│   ├── 《理解 Rust 的所有权系统》[article, intermediate, 20min]
│   ├── 《生命周期标注实战》[tutorial, intermediate, 40min]
│   └── 《Trait 与泛型：从入门到困惑到理解》[article, intermediate, 35min]
├── Day 6-8：实战项目
│   ├── 《用 Rust 写一个 CLI 工具》[project, intermediate, 120min]
│   └── 《Rust Web 开发：Actix-Web 入门》[tutorial, intermediate, 60min]
└── Day 9-10：进阶深入
    ├── 《异步 Rust：从 Future 到 Tokio》[article, advanced, 45min]
    └── 《Rust 宏编程指南》[paper, advanced, 60min]
```

**步骤 4：知识缺口检测（AI 自动）**

AI 对比路径内容与目标主题的知识图谱，检测缺口：

```
检测结果：
  ✅ 基础语法：已覆盖
  ✅ 所有权系统：已覆盖
  ⚠️ 错误处理（Result/Option）：缺少专项内容 — 建议补充
  ❌  unsafe Rust：无相关内容 — 可选进阶
  ✅ 常用 crate 介绍：已覆盖
```

系统提示用户补充缺口：可以手动搜索添加，或让 AI 推荐外部资源（标注"外部推荐"）。

**步骤 5：手动编排（用户）**

用户可以：
- 拖拽节点调整顺序
- 删除不需要的节点
- 从书签库中搜索并添加新节点
- 拆分/合并阶段（Day 分组）
- 调整每个节点的预估时间
- 重命名阶段（"Day 1-2" → "热身阶段"）

#### B.2 路径可视化

**主视图 — 路线图（Roadmap）：**

类似地铁线路图或 Notion Timeline 的纵向时间线：

- **管线**：纵向主线，按阶段分段着色
- **站点（节点）**：每个书签一个节点，显示封面缩略 + 标题 + 难度标签 + 预估时长
- **连线**：节点之间用彩色连线表示顺序，虚线表示可选节点
- **进度标记**：已完成节点用绿色填充 + 勾号，当前节点高亮脉冲
- **阶段分隔**：阶段之间用半透明分隔线 + 阶段名称标签

顶部操作栏：
- 「在图谱中查看」按钮 → 切换到知识图谱叠加视图
- 「编辑路径」按钮 → 进入拖拽编排模式
- 「导出 Markdown」按钮
- 进度条 + 统计（5/12 已完成 · 已学 3.5h · 覆盖 4 个主题）

**副视图 — 知识图谱叠加模式：**

在已有的知识图谱（参见 [[02-知识图谱视图]]）上叠加路径：

- 路径中的节点在图谱中高亮 + 编号标注（①, ②, ③...）
- 路径连线使用醒目的粗彩色箭头（不同于普通关联线的半透明细线）
- 路径之外的节点降低透明度作为背景
- 已完成的路径段用实线，未完成用虚线
- 点击编号节点 → 打开详情抽屉
- 点击「返回路线图」→ 切回主视图

**答辩演示脚本：**
> "这是一张某用户的知识图谱。切换到路径视图——可以看到 AI 推荐的 Rust 学习路线如何在用户已有的知识版图中穿行。路径不是凭空生成的，而是有机地嫁接在用户的知识结构上。"

#### B.3 进度追踪

**基础进度（A 层）：**

- 路径顶部进度条（"已完成 5/12 · 41%"）
- 每个节点复选框，点击标记完成/取消
- 标记完成时自动将书签状态更新为"已读"
- 完成整个阶段时触发阶段完成提示

**里程碑 + 激励（B 层）：**

- 每 3-5 个节点设置自动里程碑
- 完成里程碑时：
  - 庆祝动画（撒花 / 烟花粒子效果）
  - 统计卡片："这个阶段你花了 4 天 · 读了 5 篇文章 · 涵盖 3 个主题"
  - 鼓励语："你已经掌握了 Rust 的核心概念，接下来进入实战！"
- Dashboard 显示"当前学习路径进度"卡片（进度环 + 下一个待学节点）
- 每周报告中加入路径进度统计

**微型笔记链（C 层）：**

每个路径节点支持写 2-3 句微型笔记：

```typescript
interface PathNote {
  id: string;
  pathItemId: string;
  content: string;          // 笔记内容（最大 500 字）
  type: 'takeaway' | 'question' | 'todo';
  createdAt: string;
}
```

- 在路线图视图中，有笔记的节点显示一个小气泡图标
- 点击气泡 → 展开笔记内容（内联展开，不跳转）
- 所有节点的笔记串联成"笔记链"——在路径底部有一个「查看学习笔记」按钮，将所有笔记按节点顺序展开为一篇完整的阅读笔记
- 笔记类型区分：
  - 💡 **收获**（takeaway）："原来 Rust 的 borrow checker 是这样工作的..."
  - ❓ **疑问**（question）："unsafe 和 C 的指针有什么区别？待深入研究"
  - ✅ **行动**（todo）："需要动手写一个 CLI 小项目来巩固"

### 4.3 核心算法实现

#### 标签共现召回（TypeScript）

```typescript
// src/lib/recommendations/tag-cooccurrence.ts

interface ScoredBookmark {
  bookmarkId: string;
  score: number;          // 0-1，关联分数
  sharedTags: string[];   // 共享的标签名
}

async function tagCooccurrenceRecall(
  bookmarkId: string,
  limit: number = 20
): Promise<ScoredBookmark[]> {
  // 1. 获取当前书签的标签
  const bookmark = await prisma.bookmark.findUnique({
    where: { id: bookmarkId },
    include: { tags: { include: { tag: true } } },
  });
  
  const tagIds = bookmark.tags.map(t => t.tagId);
  if (tagIds.length === 0) {
    // 降级：返回同分类下的热门书签
    return await fallbackByCategory(bookmark.categoryId, limit);
  }
  
  // 2. 找到共享至少 1 个标签的所有书签
  const candidates = await prisma.bookmark.findMany({
    where: {
      id: { not: bookmarkId },
      tags: { some: { tagId: { in: tagIds } } },
    },
    include: { tags: { include: { tag: true } } },
  });
  
  // 3. 计算 Jaccard 相似度
  const scored = candidates.map(candidate => {
    const candidateTagIds = candidate.tags.map(t => t.tagId);
    const sharedTagIds = tagIds.filter(id => candidateTagIds.includes(id));
    const jaccard = sharedTagIds.length /
      (new Set([...tagIds, ...candidateTagIds]).size);
    
    return {
      bookmarkId: candidate.id,
      score: jaccard,
      sharedTags: sharedTagIds.map(id =>
        candidate.tags.find(t => t.tagId === id)?.tag.name || ''),
    };
  });
  
  // 4. 排序 + 截断
  return scored
    .sort((a, b) => b.score - a.score)
    .slice(0, limit);
}
```

#### 语义向量精排（TypeScript + AI）

```typescript
// src/lib/recommendations/semantic-rerank.ts

async function semanticRerank(
  sourceBookmarkId: string,
  candidates: ScoredBookmark[],
  topK: number = 5
): Promise<RecommendationResult[]> {
  // 1. 获取源书签的 Embedding（如果不存在则即时生成）
  let sourceEmbedding = await getEmbedding(sourceBookmarkId);
  if (!sourceEmbedding) {
    sourceEmbedding = await generateAndCacheEmbedding(sourceBookmarkId);
  }
  
  // 2. 获取候选书签的 Embedding
  const candidateEmbeddings = await Promise.all(
    candidates.map(async (c) => {
      let emb = await getEmbedding(c.bookmarkId);
      if (!emb) emb = await generateAndCacheEmbedding(c.bookmarkId);
      return { ...c, embedding: emb };
    })
  );
  
  // 3. 计算余弦相似度
  const withSimilarity = candidateEmbeddings.map(c => ({
    ...c,
    semanticScore: cosineSimilarity(sourceEmbedding, c.embedding),
  }));
  
  // 4. 混合分数：标签分数 40% + 语义分数 60%
  const mixed = withSimilarity.map(c => ({
    ...c,
    finalScore: c.score * 0.4 + c.semanticScore * 0.6,
  }));
  
  // 5. 过滤低相关 + 排序
  return mixed
    .filter(c => c.finalScore >= 0.7)
    .sort((a, b) => b.finalScore - a.finalScore)
    .slice(0, topK);
}
```

#### 余弦相似度（纯 TypeScript，不依赖外部库）

```typescript
function cosineSimilarity(a: number[], b: number[]): number {
  if (a.length !== b.length || a.length === 0) return 0;
  
  let dotProduct = 0;
  let normA = 0;
  let normB = 0;
  
  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  
  const denominator = Math.sqrt(normA) * Math.sqrt(normB);
  return denominator === 0 ? 0 : dotProduct / denominator;
}
```

### 4.6 性能考虑

| 场景 | 策略 |
|---|---|
| **Embedding 生成** | 异步后台生成，不阻塞书签创建。使用队列批量处理，避免并发 API 调用过多 |
| **标签共现计算** | 纯数据库查询 + 应用层计算，无外部依赖。1000 条书签 < 50ms |
| **语义相似度计算** | Embedding 预缓存。Top-20 候选集中计算余弦相似度，<5ms |
| **推荐响应时间** | 标签召回 + 语义精排 < 500ms（含 DB 查询 + 排序计算） |
| **路径 AI 分析** | 单次 AI 调用（批量分析所有候选书签），预计 2-5s |
| **Embedding 存储** | 1024 维 × 4 bytes = 4KB/条。10k 条书签 ≈ 40MB。SQLite 完全可承受 |
| **向量缓存** | 书签更新时删除旧 embedding，下次推荐时懒加载重新生成 |

## 五、UI/UX 设计要点

### 5.1 推荐卡片设计

- **详情页推荐卡片**：紧凑横向卡片（200×120px），封面缩略 + 匹配标签高亮 + 相似度圆环
- **Dashboard 推荐卡片**：与主站书签卡片风格一致，但带有推荐理由文案（"因为你收藏了《XXX》"）
- **发现页推荐卡片**：较大尺寸，突出封面图，带分类标签和一句话推荐理由
- **匹配标签高亮**：共享标签用主色高亮，非共享标签用灰色，直观展示关联原因

### 5.2 路线图视觉

- **管线**：纵向居中主线，每阶段配色取自 ESG 调色板（入门=绿色，进阶=蓝色，高级=紫色）
- **节点**：圆形站点，大小反映预估阅读时长。已完成 = 填充实色 + ✓，当前 = 脉冲呼吸动画，未完成 = 描边空心
- **连线**：节点间实线连接，可选节点用虚线。已完成段用绿色高亮
- **阶段标签**：左对齐的色块 + 阶段名 + 节点数
- **动画**：节点完成时缩放 + 绿色填充过渡（300ms spring）；路径切换时路线伸缩动画

### 5.3 笔记链

- **有笔记的节点**：右上角小气泡图标（💡/❓/✅），数字表示笔记数
- **展开笔记**：内联下拉展开，带浅色背景区分，不破坏路线图布局
- **笔记链面板**：底部弹出的半屏面板，所有笔记按节点顺序排列，类似聊天记录，可连续阅读
- **导出预览**：Markdown 实时预览 + 复制 / 下载按钮
