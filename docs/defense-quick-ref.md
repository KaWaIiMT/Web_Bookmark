## 快速 QA 话术

- **问：为什么用 DeepSeek 而不是 OpenAI？** → DeepSeek API 性价比更高，中文理解能力更强，且同网络环境下延迟更低。
- **问：数据库怎么选型的？** → 用 Turso（基于 libSQL 的边缘数据库），部署在 Vercel 的边缘节点附近，读写延迟低，且兼容 SQLite 语法。
- **问：前后端怎么通信的？** → Next.js App Router 的 API Routes，同仓库全栈，前端 `fetch` 调用 `/api/*`，认证通过 Cookie 中的 JWT 自动传递。
- **问：系统整体的技术亮点？** → 全栈 Next.js 16 + React 19 + TypeScript + Prisma ORM + Turso 边缘数据库 + DeepSeek AI，一键 Vercel 部署。
