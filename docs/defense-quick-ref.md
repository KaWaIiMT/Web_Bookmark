# MarkBox 答辩速查

## 分工说明

以下按难度均分 24 个核心功能，各负责 12 个。带 ⭐ 的为高分加分项。

---

## 同学 A 负责（高难度功能 — 你包揽）

| # | 功能 | 一句话实现 |
|---|------|-----------|
| 1 | **AI 智能分类与标签** ⭐ | 收藏网页时调用 DeepSeek V4 API，分析标题+描述+站点类型，返回 JSON 格式的中文标签、多级分类路径和摘要，写入数据库对应字段。 |
| 2 | **AI 多书签横向对比** ⭐ | 选定 2-5 篇书签，拼接文章信息为 Prompt 调用 DeepSeek，生成雷达图评分+对比矩阵+AI 评语三段式结构化 JSON，前端用 Recharts 渲染。 |
| 3 | **知识图谱可视化** ⭐ | 基于书签标签共现关系构建图数据，使用 react-force-graph-2d 力导向布局渲染节点和边，按类型着色、按引用数定大小，点击跳转筛选。 |
| 4 | **AI 学习路径生成** ⭐ | 用户选定书签后 AI 分析内容难度层次，自动划分"基础→进阶→实战"阶段，估算学习时长，检测知识盲区并给出补全建议。 |
| 5 | **热度追踪系统** ⭐ | 针对 GitHub 仓库书签定时抓取 Stars/Forks/Issues 数据，记录历史快照绘制 Sparkline 趋势图，自动检测并通知里程碑（如 1000 Stars）。 |
| 6 | **链接健康监控** ⭐ | 后台并行检测所有书签 URL 的 HTTP 状态，检测重定向链和死链，采用"两次确认"防误报，卡片显示绿/黄/红健康图标。 |
| 7 | **语义嵌入与关联推荐** | 调用 DeepSeek Embedding 为每篇书签生成向量，结合标签 Jaccard 相似度做混合推荐（标签40%+语义60%），详情页展示关联书签。 |
| 8 | **全文批注系统** | 阅读模式下选中文本生成锚点（xpath + offset），支持四色标记和旁注笔记，关闭后重新打开自动还原位置，支持导出 Markdown。 |
| 9 | **页面永久存档** | 对文章类书签自动抓取 HTML，cheerio 清洗脚本/样式/iframe 后存库，详情页提供存档阅读标签，支持批量刷新 30 天内书签。 |
| 10 | **对比分析客户端直连** ⭐ | 为突破 Vercel 10s 超时限制，将 DeepSeek 对比请求迁至浏览器直接调用，生成结果后 POST 到服务端保存，支持查看历史对比记录。 |
| 11 | **自动数据库迁移** | 编写 prebuild 迁移脚本，用 libSQL 直连 Turso 检测表/列是否存在并自动修复，确保每次部署 Schema 对齐，无需手动操作数据库。 |
| 12 | **API Key 加解密体系** | 生成 `mb_` 前缀的 API Key，SHA-256 哈希存库做认证，AES-256-CBC 加密存库做回显，支持命名、使用追踪、吊销，兼容旧数据。 |

## 协作方式

本项目采用 **GitHub + 功能分支 + PR Review** 的标准开源协作模式：

- **仓库托管**：代码托管在 GitHub，通过 Git 版本控制，每次提交附带清晰的 commit message 标注改动范围。
- **分支策略**：成员从 `master` 创建功能分支独立开发，完成后提交 Pull Request，另一方 Code Review 后合并，保证代码质量和知识共享。
- **Vercel 自动部署**：每次 `git push` 到 master，Vercel 自动拉取代码、运行数据库迁移脚本、构建并部署到生产环境 `ccjproject.top`，实现持续交付。
- **AI 增强协作**：引入 AI Coding Agent 作为"第三协作者"——负责 Bug 修复验证、自动数据库迁移、性能优化、代码审查等重复性高的工作。人类成员专注于架构决策和功能设计，AI 处理工程细节，形成"人定方向+AI 落地"的高效协作模式。
- **异步协作**：利用 GitHub Issues 记录需求和 Bug，PR Discussion 做设计和方案评审，不依赖同步会议即可推进项目进度。
